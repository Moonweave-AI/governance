# セキュリティとコンプライアンス

アプリケーションセキュリティ、サプライチェーンの完全性、コンプライアンスフレームワーク、脆弱性管理に関する用語集です。輝夜計画のセキュリティレベル（S0〜S5）の基礎を定義します。

---

## SBOM (Software Bill of Materials)

**日本語名：** ソフトウェア部品表（SBOM）

ソフトウェアアーティファクト内のすべてのコンポーネント、ライブラリ、依存関係の機械可読なインベントリ。脆弱性追跡、ライセンスコンプライアンス、サプライチェーンの透明性を可能にする。

## SLSA (Supply-chain Levels for Software Artifacts)

**日本語名：** SLSA（ソフトウェアアーティファクトのサプライチェーンレベル）

検証済みビルド来歴を通じてソフトウェアサプライチェーンの完全性を確保するフレームワーク。L1（基本来歴）からL4（完全隔離、再現可能ビルド）までのレベルがある。

## Provenance

**日本語名：** 来歴証明（Provenance）

誰が、どのプロセスを通じて、どのソース入力から、どの環境でアーティファクトをビルドしたかを記録する。サプライチェーン検証と監査証跡に不可欠である。

## SPDX (Software Package Data Exchange)

**日本語名：** SPDX

ソフトウェアコンポーネント情報（ライセンス、著作権、セキュリティ参照を含む）を伝達するISO標準（ISO/IEC 5962:2021）フォーマット。

## SAST (Static Application Security Testing)

**日本語名：** 静的アプリケーションセキュリティテスト（SAST）

実行なしでソースコードのセキュリティ分析を行う。インジェクション欠陥、ハードコードされたシークレット、安全でないパターンを開発中に特定する。

## DAST (Dynamic Application Security Testing)

**日本語名：** 動的アプリケーションセキュリティテスト（DAST）

攻撃をシミュレートして実行中のアプリケーションのセキュリティテストを行う。ソースコードでは見えないランタイム脆弱性を発見する。

## CodeQL

**日本語名：** CodeQL

コード構造に対するデータベーススタイルのクエリを通じて、セキュリティ脆弱性とコード品質の問題を発見するGitHubの意味論的コード分析エンジン。

## OWASP (Open Web Application Security Project)

**日本語名：** OWASP

Webアプリケーションセキュリティの標準、ツール、ガイダンスを提供する非営利団体。OWASP Top 10、ASVS、SAMMなどの主要成果物を公開している。

## ASVS (Application Security Verification Standard)

**日本語名：** ASVS（アプリケーションセキュリティ検証標準）

Webアプリケーションのセキュリティ要件の包括的チェックリストを提供するOWASP標準。検証レベル（L1〜L3）ごとに整理されている。

## OWASP SAMM (Software Assurance Maturity Model)

**日本語名：** OWASP SAMM

ガバナンス、設計、実装、検証、運用にわたるソフトウェアセキュリティ実践の評価と改善のためのフレームワーク。

## OWASP LLM Top 10

**日本語名：** OWASP LLM Top 10

大規模言語モデルアプリケーション固有の最も重大なセキュリティリスクのリスト。プロンプトインジェクション、安全でない出力処理、学習データ汚染などを含む。輝夜計画のAIエージェント開発で必須の参照資料である。

## Secret Scanning

**日本語名：** シークレットスキャン

Git履歴に誤ってコミットされた認証情報、APIキー、トークン、その他のシークレットを自動検出する。公開または共有リポジトリでの認証情報漏洩を防ぐ。

## Dependency Scanning

**日本語名：** 依存関係スキャン

プロジェクトの依存関係における既知の脆弱性（CVE）を自動検出する。DependabotやGitHub dependency reviewなどのツールがアラートと自動修正PRを提供する。

## CVE (Common Vulnerabilities and Exposures)

**日本語名：** CVE（共通脆弱性識別子）

公に知られているセキュリティ脆弱性の標準識別子（例：CVE-2024-1234）。依存関係スキャナーが既知の脆弱性をプロジェクトの依存関係と照合するために使用する。

## Threat Model

**日本語名：** 脅威モデル

システムに対する潜在的なセキュリティ脅威の構造化分析。資産、脅威アクター、攻撃ベクトル、緩和策を特定する。S3以上のシステムでは実装前に必須である。

## NIST CSF (Cybersecurity Framework)

**日本語名：** NIST CSF（サイバーセキュリティフレームワーク）

Identify、Protect、Detect、Respond、Recoverの機能にサイバーセキュリティ活動を整理するリスクベースのフレームワーク。輝夜計画のセキュリティガバナンスはCSF 2.0のリスク管理アプローチと整合している。

## NIST SSDF (Secure Software Development Framework)

**日本語名：** NIST SSDF（安全なソフトウェア開発フレームワーク）

NIST SP 800-218。準備から実装、対応、脆弱性管理まで、ソフトウェア開発ライフサイクルのすべてのフェーズにセキュリティを統合する実践を定義する。

## NIST AI RMF (AI Risk Management Framework)

**日本語名：** NIST AI RMF（AIリスク管理フレームワーク）

NIST AI 100-1。有効性、安全性、セキュリティ、説明責任、透明性、公平性、プライバシーを含む信頼性特性にわたるAIシステムリスクの管理フレームワーク。輝夜計画のAIガバナンスの参照基準である。

## OpenSSF Scorecard

**日本語名：** OpenSSF Scorecard

オープンソースプロジェクトをセキュリティヒューリスティクス（ブランチ保護、CIテスト、脆弱性開示、依存関係更新など）に対して評価し、スコアを生成する自動化ツール。

## Sigstore

**日本語名：** Sigstore

ソフトウェアアーティファクトの署名、検証、透明性ログの作成のためのツール群。リリース、コンテナイメージ、SBOMのキーレス署名を可能にする。

## Zero Trust

**日本語名：** ゼロトラスト

ユーザー、サービス、ネットワークのいずれも本質的に信頼されないセキュリティモデル。すべてのアクセス要求は、発信元に関わらず認証、認可、継続的な検証を受けなければならない。

## Supply Chain Attack

**日本語名：** サプライチェーン攻撃

アプリケーション自体ではなく、ソフトウェアサプライチェーン（依存関係、ビルドシステム、配布チャネル）を標的とする攻撃。SBOM、来歴証明、依存関係の固定、再現可能ビルドによって緩和される。
