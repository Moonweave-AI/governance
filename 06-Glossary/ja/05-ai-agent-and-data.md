# AI・エージェント・データ

人工知能、言語モデル、自律エージェント、データ管理に関する用語集です。輝夜計画のAI・エージェント領域の中核概念を定義します。

---

## Agent

**日本語名：** エージェント

環境を認知し、意思決定を行い、目標達成のために行動を取るAIシステム。輝夜計画におけるエージェントは、ツール呼び出し能力、長期記憶、パーソナリティの一貫性を持ち、明示的な権限境界のもとで動作する。

## LLM (Large Language Model)

**日本語名：** 大規模言語モデル（LLM）

大規模なテキストコーパスで学習されたニューラルネットワーク。自然言語の生成、理解、推論が可能である。会話型エージェント、コード生成、テキスト分析の計算基盤となる。

## RAG (Retrieval-Augmented Generation)

**日本語名：** 検索拡張生成（RAG）

情報検索（ナレッジベースの検索）と生成（応答の生成）を組み合わせる手法。取得した根拠に出力を接地することでハルシネーションを低減する。輝夜計画のエージェント知識基盤の中核技術である。

## Prompt

**日本語名：** プロンプト

言語モデルへの入力テキスト。モデルの応答を導く。システムプロンプト（行動指示）、ユーザープロンプト（クエリ）、few-shot例を含む。

## System Prompt

**日本語名：** システムプロンプト

エージェントの行動、能力、制約、ペルソナを定義する特別なプロンプト。すべてのインタラクションにわたる応答方法を制御する。重要な行動アーティファクトとしてバージョン管理される。

## Policy

**日本語名：** ポリシー

エージェントの行動を統治するルールの集合。許可されるアクション、拒否するトピック、不確実性への対処、人間へのエスカレーション条件を定義する。システムプロンプトとは異なり、ポリシーは境界を定義し、プロンプトは行動を定義する。

## Tool Calling

**日本語名：** ツール呼び出し

エージェントが推論プロセスの一部として外部関数（API、データベース、ファイルシステム、計算機、Web検索）を呼び出す能力。各ツールには明示的な権限付与が必要である。

## Prompt Injection

**日本語名：** プロンプトインジェクション

悪意のある入力によりAIシステムが指示を無視し、意図しないアクションを実行させる攻撃。ツールアクセスや記憶書き込み能力を持つエージェントにとって重大なセキュリティ懸念である。

## Hallucination

**日本語名：** ハルシネーション

AIモデルがもっともらしいが事実と異なる情報を生成する現象。品質と安全性の懸念であり、検出メカニズム、不確実性のシグナリング、出力検証が必要である。

## Long-term Memory

**日本語名：** 長期記憶

エージェントがセッションを超えて保持する永続的な状態。会話履歴、ユーザー設定、学習した事実、行動適応を含む。汚染とドリフトに対する完全性保護が必要である。

## Memory Poisoning

**日本語名：** 記憶汚染

攻撃者がエージェントに虚偽または悪意のある情報を長期記憶に保存させ、将来の行動に影響を与える攻撃。記憶書き込みポリシーと完全性チェックによって緩和される。

## Embedding

**日本語名：** エンベディング

テキスト、画像、その他のデータを連続空間における密なベクトル表現に変換したもの。意味的類似性が幾何的な近接性に対応する。検索、クラスタリング、類似性検索に使用される。

## Vector Store

**日本語名：** ベクトルストア

近似最近傍探索を用いて高次元エンベディングの保存とクエリに最適化されたデータベース。RAGシステムの中核インフラストラクチャである。

## Drift

**日本語名：** ドリフト

実世界データの分布が学習データから乖離するにつれ、モデル性能が時間とともに段階的に劣化する現象。監視と定期的な再学習または評価が必要である。

## Eval / Evaluation

**日本語名：** 評価（Eval）

定義された指標、データセット、行動期待値に対するAIシステム性能の体系的測定。正確性、安全性、堅牢性、公平性、コスト、レイテンシの次元を含む。輝夜計画ではデプロイ前の必須ゲートとして位置づけられる。

## Red Team

**日本語名：** レッドチーム

評価者が意図的にAIシステムに有害、不正確、またはポリシー違反の出力を生成させようとする敵対的テスト。デプロイ前に失敗モードを発見するために使用される。

## Model Card

**日本語名：** モデルカード

公開モデルに付随するドキュメントアーティファクト。想定用途、学習データ、評価結果、制限、バイアス、倫理的考慮事項を記述する。

## Dataset Card

**日本語名：** データセットカード

データセットの構成、収集プロセス、想定用途、制限、ライセンス、プライバシー考慮事項、品質チェックを記述するドキュメントアーティファクト。

## Fine-tuning

**日本語名：** ファインチューニング

専門データでの追加学習により、事前学習済みモデルを特定のタスクやドメインに適応させる。プロンプトだけでは実現できないより深いモデル行動の変更をもたらす。

## RLHF (Reinforcement Learning from Human Feedback)

**日本語名：** RLHF（人間フィードバックによる強化学習）

人間の判断を報酬信号として使用し、モデル行動を人間の嗜好に整合させる学習手法。有用性、無害性、誠実性を形成する。

## Token

**日本語名：** トークン

言語モデルのテキスト処理の基本単位。トークナイザーによって単語、部分語、文字のいずれかになりうる。コンテキストウィンドウとコストはトークン数で測定される。

## Context Window

**日本語名：** コンテキストウィンドウ

モデルが単一のフォワードパスで処理できる最大トークン数。エージェントが同時に考慮できる履歴、取得コンテキスト、指示の量を決定する。

## Inference

**日本語名：** 推論

学習済みモデルを実行して入力から出力を生成すること。フォワードパスのみであり、通常レイテンシとコストに敏感である。学習（トレーニング）とは区別される。

## Datasheets for Datasets

**日本語名：** Datasheets for Datasets（データセットのデータシート）

すべてのデータセットに、動機、構成、収集プロセス、推奨用途、配布詳細を記述するデータシートを付随させるべきであると提案するドキュメントフレームワーク。

## FAIR Principles

**日本語名：** FAIR原則

デジタル研究資産は発見可能（Findable）、アクセス可能（Accessible）、相互運用可能（Interoperable）、再利用可能（Reusable）であるべきというガイドライン。データセット、モデル、研究アーティファクトに適用される。
