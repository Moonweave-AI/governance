# ドキュメンテーションと知識管理

技術文書作成、ドキュメントシステム、知識組織化、情報アーキテクチャに関する用語集です。輝夜計画のDocs as Code方針と知識資産管理の基礎概念を定義します。

---

## Diátaxis

**日本語名：** ディアタクシス（Diátaxis）

ユーザーのニーズに基づいてコンテンツを4種類に整理するドキュメントフレームワーク。チュートリアル（学習志向）、ハウツーガイド（タスク志向）、リファレンス（情報志向）、説明（理解志向）。単一ドキュメント内での目的の混在を防ぐ。

## Docs as Code

**日本語名：** ドキュメント・アズ・コード（Docs as Code）

ドキュメントをコードと同等の厳密さで扱う哲学。Gitに保存し、PRで変更し、ピアレビューし、CIでテストし、自動デプロイする。ドキュメントがコードベースとともに進化することを保証する。輝夜計画のドキュメント標準の中核である。

## SSoT (Single Source of Truth)

**日本語名：** SSoT（信頼できる唯一の情報源）

各情報には正確に1つの権威ある場所があるという原則。他のすべての参照は、重複または矛盾するのではなく、SSoTを指し示さなければならない。

## Front Matter

**日本語名：** フロントマター

ドキュメントの先頭にある構造化メタデータ（通常YAML）。タイトル、著者、ステータス、オーナー、日付、タイプなどのプロパティを宣言する。自動化ツール、インデックス作成、ライフサイクル管理を可能にする。

## GFM (GitHub Flavored Markdown)

**日本語名：** GFM（GitHub Flavored Markdown）

標準Markdownにテーブル、タスクリスト、取り消し線、自動リンク、フェンスコードブロックを追加したGitHub拡張Markdown仕様。輝夜計画のデフォルト執筆フォーマットである。

## Mermaid

**日本語名：** マーメイド（Mermaid）

Markdownコードブロックからフローチャート、シーケンス図、クラス図などをレンダリングするテキストベースの図表言語。バイナリ画像フォーマットと異なり、バージョン管理とdiffが可能である。

## Vale

**日本語名：** Vale

Markdown、AsciiDoc、その他のテキストフォーマットに執筆スタイルルール（用語の一貫性、受動態の検出、専門用語の回避）を適用するプロースリンター。カスタムスタイルパッケージで設定可能である。

## markdownlint

**日本語名：** markdownlint

Markdownファイルの静的解析ツール。見出し構造、リストインデント、行長、空行、コードブロックフォーマットなど一貫したフォーマットルールを適用する。

## Technical Radar

**日本語名：** テクノロジーレーダー

ThoughtWorksに由来する可視化。テクノロジーを4つのリングに分類する。Adopt（推奨デフォルト）、Trial（出口パス付きの限定使用）、Assess（研究のみ）、Hold（導入禁止）。テクノロジーガバナンスに使用される。

## Changelog

**日本語名：** 変更履歴（Changelog）

プロジェクトの各バージョンの注目すべき変更を記録するファイル（通常`CHANGELOG.md`）。ユーザーとメンテナー向け（Gitログとは異なり、開発者向けではない）。

## Release Notes

**日本語名：** リリースノート

リリースに付随するユーザー向けドキュメント。重要な変更、破壊的変更、移行手順、既知の問題をわかりやすい言葉でハイライトする。

## Glossary

**日本語名：** 用語集（Glossary）

プロジェクト全体で一貫した理解を確保するための用語定義の集合。本ドキュメント自体が輝夜計画の用語集システムの一部である。

## DOI (Digital Object Identifier)

**日本語名：** DOI（デジタルオブジェクト識別子）

デジタルオブジェクト（論文、データセット、ソフトウェアバージョン）の永続的識別子。URLの変更に関わらず長期的な発見可能性と引用可能性を保証する。

## REUSE

**日本語名：** REUSE

FSFEによる仕様。SPDX識別子を使用して、ファイルレベルでソフトウェアライセンスと著作権情報を機械可読かつ明確にする。

## Quality Declaration

**日本語名：** 品質宣言（Quality Declaration）

コンポーネントが品質レベルを宣言し、テスト、ドキュメント、バージョンポリシー、依存関係にわたってそのレベルの要件を満たすエビデンスを提供する文書。ROS 2 REP-2004に触発されている。輝夜計画のQA-L2以上のコンポーネントで要求される。

## Runbook

**日本語名：** ランブック

システム管理、インシデント対応、復旧のためのステップバイステップ手順を提供する運用ドキュメント。ユーザードキュメントとは異なり、時間的プレッシャーの下で作業するオペレーター向けである。

## Knowledge Asset

**日本語名：** 知識資産（Knowledge Asset）

理解、保守、再現、監査、作業の継続を可能にする記録された知識。コード、ドキュメント、意思決定、実験、データ記録、運用手順を含む。輝夜計画ではすべての知識資産にオーナーシップとライフサイクル管理を適用する。
