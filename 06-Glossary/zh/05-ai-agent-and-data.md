# AI、Agent 与数据

本文档定义 Moonweave/辉夜计划（Kaguya Project）治理文档中使用的人工智能、语言模型、自主 Agent 与数据管理相关术语。

---

## Agent

**中文名称：** 智能体（Agent）

能够感知环境、做出决策并采取行动以达成目标的 AI 系统。在辉夜计划中，Agent 具备工具调用、长期记忆与人格一致性，并在明确的权限边界内运行；对应 QA-L4 质量等级。

## LLM (Large Language Model)

**中文名称：** 大语言模型（LLM）

在海量文本语料上训练的神经网络，可生成、理解并推理自然语言。是对话 Agent、代码生成与文本分析的算力基础；辉夜计划 Agent 栈的核心推理引擎。

## RAG (Retrieval-Augmented Generation)

**中文名称：** 检索增强生成（RAG）

结合信息检索（搜索知识库）与生成（产出回答）的技术。通过将输出锚定于检索证据以降低幻觉；是辉夜计划 Agent 知识问答与文档理解的主要架构模式。

## Prompt

**中文名称：** 提示词

提供给语言模型以引导其响应的输入文本。包括系统提示词（行为指令）、用户提示词（查询）与 few-shot 示例。Prompt 设计直接影响 Agent 行为，须纳入版本控制与审查。

## System Prompt

**中文名称：** 系统提示词

定义 Agent 行为、能力、约束与人格的特殊提示词。控制 Agent 在所有交互中的响应方式，作为关键行为制品须版本控制；变更须经过与代码同等级别的审查流程。

## Policy

**中文名称：** 策略

规范 Agent 行为的规则集：允许哪些动作、拒绝哪些话题、如何处理不确定性、何时升级至人工。与系统提示词不同——策略定义边界，提示词定义行为。辉夜计划 Agent 须同时配置 Policy 与 System Prompt。

## Tool Calling

**中文名称：** 工具调用

Agent 在推理过程中调用外部函数（API、数据库、文件系统、计算器、网络搜索等）的能力。每个工具须显式授权；是 Prompt Injection 与权限越界的主要攻击面，须受 Policy 与沙箱约束。

## Prompt Injection

**中文名称：** 提示注入

恶意输入导致 AI 系统忽略指令并执行非预期行为的攻击。对具备工具访问或记忆写入能力的 Agent 尤为关键；辉夜计划参照 OWASP LLM Top 10 对此类风险进行 Red Team 测试。

## Hallucination

**中文名称：** 幻觉

AI 模型生成听起来合理但事实错误的信息。是质量与安全关切，需要检测机制、不确定性信号与输出验证。RAG 与 Eval 是辉夜计划缓解幻觉的主要手段。

## Long-term Memory

**中文名称：** 长期记忆

Agent 跨会话保留的持久状态——对话历史、用户偏好、习得事实与行为适应。须防范投毒与漂移，对完整性保护有明确要求；写入策略须与 Policy 一致。

## Memory Poisoning

**中文名称：** 记忆投毒

攻击者使 Agent 在长期记忆中存储虚假或恶意信息，影响后续行为的攻击。通过记忆写入策略与完整性检查缓解；辉夜计划 Agent 记忆层须记录写入来源并支持审计。

## Embedding

**中文名称：** 嵌入 / 向量表示

文本、图像或其他数据在连续空间中的稠密向量表示，语义相似性对应几何邻近性。用于检索、聚类与相似度搜索，是 RAG 与向量存储的基础。

## Vector Store

**中文名称：** 向量数据库

针对存储与查询高维嵌入进行优化的数据库，使用近似最近邻搜索。RAG 系统的核心基础设施；辉夜计划 Agent 知识库基于向量存储实现语义检索。

## Drift

**中文名称：** 漂移

随真实世界数据分布与训练数据逐渐偏离，模型性能渐进退化的现象。需要监控与定期重训练或 Eval；辉夜计划对生产 Agent 须建立 Drift 监控与再评估流程。

## Eval / Evaluation

**中文名称：** 评测（Eval）

针对定义好的指标、数据集与行为期望，对 AI 系统性能进行系统性测量。涵盖准确性、安全、鲁棒性、公平、成本与延迟等维度；QA-L4 组件发布前的必备证据。

## Red Team

**中文名称：** 红队测试

评估者刻意尝试使 AI 系统产生有害、错误或违反策略输出的对抗性测试。用于部署前发现失效模式；辉夜计划 Agent 在达到 QA-L4 前须完成 Red Team 评估。

## Model Card

**中文名称：** 模型卡

随发布模型附带的文档制品，描述预期用途、训练数据、评测结果、局限、偏见与伦理考量。辉夜计划对外或共享模型须发布 Model Card。

## Dataset Card

**中文名称：** 数据集卡

描述数据集组成、采集流程、预期用途、局限、许可、隐私考量与质量检查的文档制品。与 Datasheets for Datasets 框架一致，是辉夜计划数据资产治理的基本要求。

## Fine-tuning

**中文名称：** 微调

通过在专用数据上进一步训练，使预训练模型适应特定任务或领域。比单纯 Prompt 更深层地改变模型行为；辉夜计划微调须记录数据血缘并完成 Eval。

## RLHF (Reinforcement Learning from Human Feedback)

**中文名称：** 基于人类反馈的强化学习（RLHF）

以对齐模型行为与人类偏好为目标的训练技术，以人类判断作为奖励信号。塑造有用性、无害性与诚实性；理解 RLHF 有助于评估基础模型行为边界。

## Token

**中文名称：** 令牌（Token）

语言模型文本处理的基本单位。根据分词器，一个 Token 可以是词、子词或字符。上下文窗口与成本均以 Token 计量；Agent 设计须考虑 Token 预算与截断策略。

## Context Window

**中文名称：** 上下文窗口

模型在单次前向传播中可处理的最大 Token 数。决定 Agent 可同时考虑多少历史、检索上下文与指令；RAG 与长期记忆架构须在此约束内设计。

## Inference

**中文名称：** 推理

运行已训练模型以从输入产生输出。与训练不同，仅做前向传播且通常对延迟/成本敏感。辉夜计划 Agent 服务的 SLO 与成本模型主要围绕 Inference 优化。

## Datasheets for Datasets

**中文名称：** 数据集说明书

文档框架，主张每个数据集应附带说明书，描述动机、组成、采集流程、推荐用途与分发细节。辉夜计划 Dataset Card 与此框架对齐。

## FAIR Principles

**中文名称：** FAIR 原则

数字研究资产应可发现（Findable）、可访问（Accessible）、可互操作（Interoperable）、可重用（Reusable）的指南。应用于数据集、模型与研究制品；辉夜计划研究产出遵循 FAIR 原则以支持长期复用。
