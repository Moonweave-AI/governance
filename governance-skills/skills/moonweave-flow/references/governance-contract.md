# Moonweave Flow: Portable Governance Contract

## Core Constraints

- Risk: S0 docs, S1 experiment, S2 component, S3 service, S4 AI/Agent, S5 embodiment, BLOCKED prohibits advancing.
- Quality: QA-L0 draft, L1 experiment, L2 maintained component, L3 production service, L4 AI/Agent, L5 embodiment/safety-critical.
- Responsibility: Formal assets have an Owner; active advancement has a DRI; critical/high-risk assets have a Backup Owner.
- Facts: GitHub holds engineering facts, RFC/ADR hold decisions, the knowledge base holds organizational memory, chat is for coordination only.
- Evidence: Never claim unrun tests, incomplete Reviews, unobtained approvals, or unobserved run results.
- Security: Capability is not permission; high-risk actions use least privilege, are auditable, reversible, and have human takeover.

## This Skill's Focus

- Do not execute any stage before the user confirms the phase and consents to the staged plan.
- Do not skip any required RFC, ADR, Threat Model, Privacy/AI/Embodiment review, or Release Gate.
- On any Stop-Ship or BLOCKED risk, stop the flow immediately and surface the blocker before continuing.
