# Organization and Collaboration Semantic Core

- Responsibility before titles; permissions must correspond to a clear scope, contribution record, and revocable trust.
- Distributed autonomy, centralized convergence; daily decisions are handled by the in-scope Owner/Maintainer, cross-domain major changes go through RFC, and final appeal goes to the governance body.
- Formal assets must have a Primary Owner; high-risk/critical assets also need a Backup Owner.
- Plans have an Owner, current execution has a DRI; chat is not a task system, and important conclusions must have an authoritative GitHub/RFC/ADR/knowledge-base link.
- GitHub carries engineering fact; the knowledge base carries organizational memory; chat and meetings carry fast coordination; Agents only remind, route, summarize, and draft — they do not adjudicate.
- Contributions include code, documentation, research, evaluation, security, data, design, embodied work, community, and governance work.
