# Moonweave ガバナンス意味論コア：基盤

## 譲歩できない原則

1. **資産の清浄性と出典の追跡可能性**：コード、データ、モデル、メディア、Prompt、ファームウェア、バイナリには出典、ライセンス、バージョンまたはハッシュ、Owner、許可用途、制限が必須。
2. **全体の可観測性と振る舞いの追跡可能性**：主要な入力、状態、バージョン、ツール呼び出し、出力、評価記録は、監査・局在化・妥当な再再生に十分であること。
3. **身体性の収束とフェイルセーフ**：物理実行権限は個別に承認される必要がある。デフォルトで境界、等級付き承認、監視可能、中止可能、フェイルセーフにより制約される。
4. **状態の持続と実装の交換可能性**：モデル、フレームワーク、言語、サービスはツールである。長期状態、意味の連続性、進化の歴史は、一時的な実装に固定されてはならない。
5. **統一ドメイン契約、分散実装**：リポジトリ、フロント/バックエンド、Agent、Infra、身体性の境界を越えて権威ある概念、Schema、プロトコル、状態、互換性戦略を共有。内部実装はドメイン制約により異なってよい。
6. **寿命で構築、品質は毎回の変更に組み込む**：実験、共有コンポーネント、生産基盤は異なる閾値を採用。通常の変更が全体の健全性を理由なく低下させてはならない。
7. **伝達可能なシステムが個人英雄エンジニアリングより優先**：重要な知識、権限、保守責任、意思決定は他人が引き継げる必要がある。
8. **デフォルトで透明、可能なら公開**：安全、プライバシー、法律、IP、第三者義務が許す場合に公開。公開資産にはライセンス、出典、状態、保守責任が必須。

## コンフリクト裁定

法律/安全/プライバシー/倫理の底線 → ミッションの整合性 → 実影響 → 最小の爆発半径と可逆性 → DRI裁定 → RFC/ADRまたはIssue/PRの記録。
