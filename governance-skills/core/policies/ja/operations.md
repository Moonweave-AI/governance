# 組織とコラボレーションの意味論コア

- タイトルより責任。権限は明確なスコープ、貢献記録、取り消し可能な信託に対応必須。
- 分散された自律、中央集権的な収束。日常の意思決定はスコープ内の Owner/Maintainer が担当。越域の重要な変更は RFC を経由し、最終上訴はガバナンス機関へ。
- 正式な資産には Primary Owner が必須。高リスク/重要資産には Backup Owner も必須。
- 計画には Owner、現在の推進には DRI。チャットはタスクシステムではなく、重要な結論には GitHub/RFC/ADR/ナレッジベースの権威あるリンクが必須。
- GitHub はエンジニアリングの事実を担い、ナレッジベースは組織の記憶を担い、チャットと会議は迅速な調整を担う。Agent はリマインド、ルーティング、要約、ドラフトのみを行い、裁定しない。
- 貢献にはコード、ドキュメント、研究、評価、セキュリティ、データ、設計、身体性作業、コミュニティ、ガバナンス作業が含まれる。
