# 知識資産の意味論コア

- 重要な知識点はそれぞれ権威ある事実源を一つだけ持つ。他の場所は参照、要約、索引のみ。
- 正式なドキュメントは Docs as Code を採用：Git、PR、Review、自動チェック、追跡可能な履歴。
- 正式なドキュメントにはタイプ、Owner、状態、対象読者、更新時刻、可視性、ライフサイクルが必須。
- Tutorial は学習用、How-to はタスク完了用、Reference は事実参照用、Explanation は背景理解用。混在させない。
- RFC は「そうすべきか、なぜか」に答え、ADR は「何が決定され、その帰結」に答え、Design Doc は「どう実装するか」に答え、PR は実際の変更を記録する。
- 研究記録には問題、仮説、コード/データ/モデル/設定/ハードウェア/乱数シード、手法、指標、生結果、ネガティブ結果、限界、再現方法、結論が必須。
