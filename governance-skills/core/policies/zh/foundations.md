# 辉夜计划治理语义内核：根基

## 不可丢失的原则

1. **资产清洁与来源可追溯**：代码、数据、模型、媒体、Prompt、固件和二进制必须有来源、许可、版本或哈希、Owner、允许用途与限制。
2. **全局可观测与行为可追踪**：关键输入、状态、版本、工具调用、输出和评估记录必须足以审计、定位或合理重放。
3. **具身收敛与失效安全**：物理执行权必须单独授权；默认受边界、分级授权、可监督、可中止和失效安全约束。
4. **状态持存与实现可替换**：模型、框架、语言和服务是工具；长期状态、语义连续性和演进历史不得被某个临时实现锁死。
5. **统一领域契约，分布式实现**：跨仓库、前后端、Agent、Infra、具身边界共享权威概念、Schema、协议、状态和兼容策略；内部实现可因领域约束不同。
6. **按寿命构建，质量内建于每次变更**：实验、共享组件、生产底座采用不同门槛；正常变更不得无说明降低整体健康度。
7. **可传承系统优于个人英雄工程**：关键知识、权限、维护责任和决策必须可被他人继承。
8. **默认透明，能开则开**：在安全、隐私、法律、知识产权和第三方义务允许时开放；公开资产必须有许可、来源、状态和维护责任。

## 冲突裁决

法律/安全/隐私/伦理底线 → 使命对齐 → 真实影响 → 最小爆炸半径与可逆性 → DRI裁决 → RFC/ADR或Issue/PR留痕。
