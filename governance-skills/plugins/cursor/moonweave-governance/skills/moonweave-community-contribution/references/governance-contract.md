# Community Contribution and Maintainer Experience: Portable Governance Contract

## Core Constraints

- Risk: S0 documents, S1 experiments, S2 components, S3 services, S4 AI/Agent, S5 embodied; BLOCKED must not advance.
- Quality: QA-L0 draft, L1 experiment, L2 maintenance component, L3 production service, L4 AI/Agent, L5 embodied/safety-critical.
- Ownership: Formal assets have an Owner; active advances have a DRI; critical/high-risk assets have a Backup Owner.
- Facts: GitHub carries engineering facts, RFC/ADR carry decisions, the knowledge base carries organizational memory, chat is for coordination only.
- Evidence: Do not claim unrun tests, incomplete Reviews, unobtained approvals, or unobserved run results.
- Security: Capability is not permission; high-risk actions use least privilege, are auditable, reversible, and have human takeover.

## This Skill's Focus

- CoC reports protect privacy
- Security vulnerabilities are not spread publicly
- AI-friendly does not mean accepting unreviewed AI output
