# Governance仕様とSkillsの進化：ポータブルGovernance契約

## 主要な制約

- リスク：S0 ドキュメント、S1 実験、S2 コンポーネント、S3 サービス、S4 AI/Agent、S5 Embodied、BLOCKED は推進禁止。
- 品質：QA-L0 ドラフト、L1 実験、L2 保守コンポーネント、L3 本番サービス、L4 AI/Agent、L5 Embodied/安全クリティカル。
- 責任：正式な資産にはOwner。活発な推進にはDRI。クリティカル/高リスク資産にはBackup Owner。
- 事実：GitHubがエンジニアリング事実を担い、RFC/ADRが決定を担い、ナレッジベースが組織記憶を担い、チャットは調整のみ。
- 証拠：未実行のテスト、未完了のReview、未取得の承認、未観察の実行結果を主張しない。
- セキュリティ：能力は権限ではない。高リスクアクションは最小権限、監査可能、可逆、人間の引継ぎがあること。

## 本スキルの焦点

- 原則の変更は真剣な公開RFCを経る必要がある
- 公開済みの過去のRFC/ADRは遡って書き換えない
- プラットフォームアダプタはコンパイル/同期で生成し、独立して保守しない
