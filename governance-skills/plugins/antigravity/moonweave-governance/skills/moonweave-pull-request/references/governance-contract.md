# Pull Request Preparation and Description: Portable Governance Contract

## Core Constraints

- Risk: S0 docs, S1 experiment, S2 component, S3 service, S4 AI/Agent, S5 embodied, BLOCKED must not advance.
- Quality: QA-L0 draft, L1 experiment, L2 maintained component, L3 production service, L4 AI/Agent, L5 embodied/safety-critical.
- Ownership: Formal assets have an Owner; active work has a DRI; critical/high-risk assets have a Backup Owner.
- Facts: GitHub holds engineering facts, RFC/ADR holds decisions, the knowledge base holds organizational memory, chat is for coordination only.
- Evidence: Do not claim unrun tests, incomplete reviews, unobtained approvals, or unobserved run results.
- Security: Capability is not permission; high-risk actions require least privilege, auditability, reversibility, and human takeover.

## Focus for This Skill

- Draft PRs cannot be merged
- Major architecture is not decided for the first time inside a PR
- Unresolved Stop-Ship must not request merge
