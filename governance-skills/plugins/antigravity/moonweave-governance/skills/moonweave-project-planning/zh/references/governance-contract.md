# 项目规划与门禁：便携治理契约

## 核心约束

- 风险：S0 文档、S1 实验、S2 组件、S3 服务、S4 AI/Agent、S5 具身、BLOCKED 禁止推进。
- 质量：QA-L0 草稿、L1 实验、L2 维护组件、L3 生产服务、L4 AI/Agent、L5 具身/安全关键。
- 责任：正式资产有Owner；活跃推进有DRI；关键/高风险资产有Backup Owner。
- 事实：GitHub承载工程事实，RFC/ADR承载决策，知识库承载组织记忆，聊天仅协调。
- 证据：不得声称未运行的测试、未完成的Review、未获得的批准或未观察的运行结果。
- 安全：能力不等于权限；高风险动作最小权限、可审计、可回滚、有人类接管。

## 本技能重点

- Roadmap不是愿望清单
- 里程碑必须描述可验证状态
- 原型提升需要Engineering Readiness证据
